Theory
Understand the concept fully before you try it in Excel
Why formulas exist

Imagine you have 120 rows of FMCG sales data — 10 brands, 12 months, 5 regions. Your manager asks: "What was the total revenue last year?" Without formulas, you would have to add 120 numbers manually — that takes minutes and introduces human error. With = SUM(I4:I123), you get the answer in 2 seconds. That is the power of formulas.

Every formula in Excel starts with an equals sign =. Without it, Excel treats what you type as plain text, not a calculation. The equals sign is Excel's signal that you are about to give it an instruction, not type a label.

A range like I4:I123 means "all cells from I4 down to I123" — that colon : means "through". When you select a column of data in Excel, you are selecting a range. Ranges are the inputs to every formula you will write.

Important: Row 3 in your FMCG dataset is the header row (Month, Brand, Region, Revenue etc.). Your actual data starts from row 4 and goes to row 123. That is why all formulas use ranges starting at row 4, not row 3.
The five core functions
SUM
=SUM(range)
Adds all numbers in the range. Empty cells and text are ignored automatically — no special handling needed.
=SUM(I4:I123)
→ Total revenue across all 120 rows
AVERAGE
=AVERAGE(range)
Calculates the arithmetic mean: adds all numbers and divides by how many non-empty cells there are.
=AVERAGE(I4:I123)
→ Mean revenue per row
MIN
=MIN(range)
Scans the entire range and returns only the single smallest value. Ignores text and empty cells completely.
=MIN(I4:I123)
→ Lowest single revenue value
MAX
=MAX(range)
Returns only the single largest value in the range. The opposite of MIN. Together they show the spread of your data.
=MAX(I4:I123)
→ Highest single revenue value
COUNT / COUNTA
=COUNT(range)     =COUNTA(range)
COUNT counts only cells that contain numbers. COUNTA counts any non-empty cell — numbers, text, dates. Use COUNT on number columns, COUNTA on text columns like Brand or Region. If COUNT returns less than 120, you have missing revenue values — a data quality problem you need to investigate.
=COUNT(I4:I123)   → counts number cells     =COUNTA(D4:D123)   → counts brand name cells
→ Both should return 120 if your data has no blanks
How a range works — from your cell address to a formula result
Understanding the range — I4:I123 explained

In your FMCG Sales dataset, the columns are: A=Row_ID, B=Month, C=Month_Num, D=Brand, E=Category, F=Region, G=Units_Sold, H=Unit_Price, I=Revenue, J=Target, K=Achievement_%, L=YoY_Growth_%.

Row 3 is your header row. Data starts at row 4. You have 120 data rows (10 brands × 12 months), so data ends at row 123. That is why Revenue range is I4:I123 and Units_Sold range is G4:G123. If you ever add more rows, just change 123 to however many rows you have.

Common mistake: Including the header row in your range. If you write =SUM(I3:I123), Excel will try to add "Revenue" (the header text) plus all the numbers. Excel ignores the text, so the result is technically correct — but it is sloppy practice. Always start ranges at row 4.

Your scenario
Your manager has asked for a complete data summary of the FMCG Sales dataset. She needs numbers for both the Revenue column (I) and the Units Sold column (G). Build two separate summary blocks — one for revenue, one for units. All five metrics for each column. Use only Excel formulas — no typed numbers.
Question 1
What is the total units sold across all 120 rows?
Write the SUM formula for the Units_Sold column (column G, rows 4 to 123)
Units_Sold is in column G. Your range starts at G4 and ends at G123. The function is SUM. Pattern: =SUM(G4:G123)
Question 2
What is the average unit price across all rows?
Unit_Price is in column H. Write the AVERAGE formula.
Unit_Price is in column H. Range: H4:H123. Function: AVERAGE. Pattern: =AVERAGE(H4:H123)
Question 3
What is the minimum revenue in any single row? And the maximum?
Write two separate formulas — one MIN, one MAX — both on the Revenue column (I)
MIN formula: =MIN(I4:I123) — this gives the lowest revenue row. MAX formula: =MAX(I4:I123) — this gives the highest revenue row. Two separate cells, two separate formulas.
Question 4
How many rows have revenue data? And how many rows have a brand name?
Use COUNT on the Revenue column and COUNTA on the Brand column (D). Both should return 120 if data is complete.
COUNT on numbers: =COUNT(I4:I123) — counts numeric cells. COUNTA on text: =COUNTA(D4:D123) — counts any non-empty cell. Brand names are text so you need COUNTA not COUNT.
Question 5 — Bonus
Write a single formula that calculates the revenue range (the difference between the highest and lowest revenue values)
This is MAX minus MIN in one formula. You do not need a separate cell for each.
Combine MAX and MIN in one formula: =MAX(I4:I123)-MIN(I4:I123). This subtracts the minimum from the maximum in a single cell. No intermediate cells needed.

Q1 — Total units sold
Correct
=SUM(G4:G123)
SUM adds every number in the range G4:G123. Column G is Units_Sold. Row 4 is the first data row (row 3 is the header). Row 123 is the last data row. The colon between G4 and G123 means "all cells from G4 through G123 inclusive" — that is 120 cells.
Why not =SUM(G:G)? That would include the header cell G3 which says "Units_Sold" — Excel ignores text in SUM so the result is the same, but it is bad practice. Always use precise ranges.
Q2 — Average unit price
Correct
=AVERAGE(H4:H123)
AVERAGE adds all values in H4:H123 and divides by the count of non-empty cells. Column H is Unit_Price. The result tells you the typical price per unit across all 120 sales rows — a useful benchmark for pricing analysis.
Important: AVERAGE ignores empty cells automatically. If 10 of your 120 rows have no price, AVERAGE divides by 110, not 120. This is usually what you want — but always check COUNT to know how many rows actually have data.
Q3 — Minimum and Maximum revenue
Correct
=MIN(I4:I123) =MAX(I4:I123)
MIN scans all 120 revenue values and returns only the single smallest one. MAX returns only the largest. These two numbers together define the spread of your revenue data. A large gap between MIN and MAX means high variability — some brands or months perform very differently from others.
Real-world use: MIN helps you find your worst-performing month for investigation. MAX helps you identify what conditions led to your best performance, so you can replicate them.
Q4 — Count rows
Correct
=COUNT(I4:I123)     → counts number cells in Revenue =COUNTA(D4:D123)   → counts text cells in Brand
COUNT only counts cells containing numbers. Revenue values are numbers so COUNT works. Brand names are text — COUNT would return 0 for them. COUNTA counts any non-empty cell regardless of type, so it works for text columns like Brand, Region, Month.
If COUNT returns 118 but COUNTA returns 120, it means 2 revenue cells are blank even though those rows have a brand name. That is a data quality problem — you have rows with no revenue recorded.
Q5 Bonus — Revenue range in one formula
Correct
=MAX(I4:I123)-MIN(I4:I123)
You can nest functions inside each other in a single formula. Excel evaluates MAX(I4:I123) first, then MIN(I4:I123), then subtracts. The result is the total spread of your revenue — the difference between your best and worst performing row. This is called the data range in statistics.
This is the start of more complex formulas. As you progress through the course you will combine three, four, even five functions in a single formula. The skill of nesting functions is what separates beginner analysts from advanced ones.

The 5 most common mistakes in this lesson
!
Forgetting the equals sign
If you type SUM(I4:I123) without the = sign, Excel displays it as text, not a formula. The cell will literally show the words "SUM(I4:I123)" instead of a number. The = is not optional — it tells Excel "this is a formula, not a label".
Wrong: SUM(I4:I123)    Correct: =SUM(I4:I123)
#
Using the wrong column letter
If you write =SUM(G4:G123) when you meant Revenue (column I), you will get the total of Units_Sold instead. Always click on a cell in the column you want to verify which letter it is — look at the column header or the Name Box (top left) which shows the cell address.
Units = G   Price = H   Revenue = I   Target = J   Achievement% = K
:
Missing the colon between cell references
The colon tells Excel "from cell A to cell B". Without it, =SUM(I4 I123) is a syntax error. Some people accidentally type a comma instead of a colon — =SUM(I4,I123) only adds those two specific cells, not the whole range between them.
Wrong: =SUM(I4,I123) → adds only 2 cells   Correct: =SUM(I4:I123) → adds all 120
3
Starting range at row 3 instead of row 4
Row 3 is your header row — it contains the text "Revenue". If you write =SUM(I3:I123), Excel includes the header. Since "Revenue" is text, SUM ignores it and the result is technically the same — but it is imprecise and will cause problems with other functions like COUNT.
Wrong: =SUM(I3:I123)   Correct: =SUM(I4:I123)
C
Using COUNT on a text column
COUNT only counts cells with numbers. If you use =COUNT(D4:D123) on the Brand column (which contains text like "HUL", "Dabur"), you will get 0 — not 120. For text columns always use COUNTA, which counts any non-empty cell.
COUNT on text → 0 (wrong)   COUNTA on text → 120 (correct)
